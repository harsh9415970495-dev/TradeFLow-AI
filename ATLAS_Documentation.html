01 · Project Overview
ATLAS AI Trading & Learning Analytics System क्या है?

ATLAS AI Trading & Learning Analytics System एक AI-Powered Virtual Stock Trading Platform है जो Indian Stock Market (NSE) पर simulate trading की सुविधा देती है। हर user को ₹10,00,000 virtual cash मिलते हैं जिससे वो real market data (Yahoo Finance API) पर trade practice कर सकता है — बिना कोई real money risk किए।

Purpose: India में millions लोग stock market में invest करना चाहते हैं लेकिन practice करने का safe environment नहीं है। ATLAS AI Trading & Learning Analytics System वो risk-free environment provide करता है जहाँ users real-time market data के साथ trade करें, AI insights से सीखें, और confident investor बनें।

07 · Mathematical Calculations & Formulas
Core Trading Mathematics

1. Average Buy Price (Weighted Average)

Jab user same stock ke multiple BUY orders place karta hai, to average price calculate hota hai:

New Avg Price = ((Old Avg × Old Qty) + (Buy Price × New Qty)) / (Old Qty + New Qty)

Example: 10 shares @ ₹100 + 5 shares @ ₹120 = ((100×10) + (120×5)) / (10+5) = ₹106.67

Code Location: orderController.js line 54, liveMarketFetcher.js line 25

2. Profit & Loss (P&L)

Investment Value = Quantity × Average Buy Price Current Value = Quantity × Current Market Price Total P&L = Current Value - Investment Value P&L Percentage = (Total P&L / Investment Value) × 100

Code Location: portfolioController.js lines 49-69

3. Today's P&L

Today's P&L per stock = Quantity × Day Change (₹) Total Today P&L = Sum of all holdings' today P&L

Day Change = stock.change (currentPrice - previousClose). Calculated in liveMarketFetcher.js.

Code Location: portfolioController.js line 52

4. Day Change & Change Percent

Change = Current Price - Previous Close Change % = (Change / Previous Close) × 100

Yahoo Finance API se regularMarketPrice aur chartPreviousClose milta hai, unmein se calculate hota hai.

Code Location: liveMarketFetcher.js lines 101-102

5. Total Portfolio Value

Total Portfolio Value = Cash Balance + Sum(Current Value of all Holdings) Performance % = ((Total Holdings Value - Total Investment) / Total Investment) × 100

Code Location: portfolioController.js lines 75-89

6. Market Order Execution

BUY: Total Cost = Quantity × Current Price → Deduct from cashBalance SELL: Total Credits = Quantity × Current Price → Add to cashBalance

Code Location: orderController.js lines 36, 113

7. Limit Order Trigger Logic

BUY Limit: Triggered when Stock Price ≤ Limit Price SELL Limit: Triggered when Stock Price ≥ Limit Price

Background worker har 30 seconds mein check karta hai. Triggered hone par same MARKET order logic execute hota hai.

Code Location: liveMarketFetcher.js lines 9-10

8. AI Mock RSI & SMA Fallback

Mock RSI = 45 + Random(0-20) [Range: 45-65] SMA(20) = Sum(last 20 closing prices) / min(20, data_length) Trend = Current Price > SMA ? "Uptrend" : "Downtrend"

Jab Groq AI fail hota hai, backend locally ye mock calculations use karta hai.

Code Location: aiController.js lines 98-100

9. Market Mood Calculation

Gainers = Stocks with changePercent > 0 Losers = Stocks with changePercent ≤ 0 If Gainers > Losers + 2 → "Bullish" If Losers > Gainers + 2 → "Bearish" Otherwise → "Neutral"

Code Location: aiController.js lines 199-204

09 · Real-Time System
Socket.io + Yahoo Finance Live Data

Yahoo Finance Data Fetcher (liveMarketFetcher.js)

API Endpoint: https://query1.finance.yahoo.com/v8/finance/chart/{SYMBOL}?interval=1d&range=1d

Polling Interval: Every 30 seconds

Rate Limit Protection: Stocks sequentially process hote hain with 1000ms delay between each request (50 stocks ≈ 50 seconds). Ye Yahoo Finance rate limits se bachata hai.

Data Extracted: regularMarketPrice, chartPreviousClose, regularMarketVolume

After Each Price Update:

  • Database mein stock price update
  • Socket.io broadcast via stockPriceUpdate event
  • Check all PENDING limit orders → auto-execute if triggered
  • Check all price alerts → trigger notification if condition met

Socket.io Rooms & Events

Room/EventTypeDescription

orderExecutedUser-SpecificTriggered when a limit order gets auto-executed

10 · Data Flow & Workflow
End-to-End User Journey

User Registration Flow

1. User fills Register form → POST /api/auth/register
2. Backend: Check duplicate email/username
3. Hash password with bcrypt (salt = 10)
4. Create User in Supabase (cashBalance = ₹10,00,000)
5. Generate JWT token (expires: 30 days)
6. Frontend: Store token in localStorage
7. Set Axios default Authorization header
8. Redirect to Dashboard

Market BUY Order Flow

1. User clicks "BUY" on Stock Details → Opens OrderModal
2. User sets quantity, order type (MARKET) → Submit
3. Frontend: POST /api/orders/place {symbol, type:BUY, orderType:MARKET, quantity}
4. Backend:
   a. Find stock → get currentPrice
   b. Calculate totalCost = quantity × currentPrice
   c. Check: cashBalance ≥ totalCost ?
   d. Deduct cashBalance
   e. Find/Create Portfolio & Holding
   f. Update Holding (weighted avg price if existing)
   g. Create Transaction record
   h. Create Order record (status: EXECUTED)
5. Response: {success, message, order, cashBalance}
6. Frontend: updateBalance() → refreshPortfolio() → close modal

Limit Order Auto-Execution Flow

1. User places LIMIT BUY @ ₹100 → Order created (status: PENDING)
2. Background: liveMarketFetcher runs every 30 seconds
3. For each stock, fetch live price from Yahoo Finance
4. After DB update → checkLimitOrders(stock) is called
5. Find all PENDING orders for this stock
6. If BUY LIMIT and currentPrice ≤ limitPrice → TRIGGER
7. Execute same logic as MARKET order
8. Update order status to EXECUTED
9. Create Notification in DB
10. Socket.io → sendToUser('orderExecuted', {...})
11. Socket.io → sendToUser('notification', {...})

Live Price Update Flow

1. Backend: liveMarketFetcher starts on server boot
2. Every 30 seconds, loop through all 50 stocks
3. Fetch Yahoo Finance API (with 1s delay between stocks)
4. Extract: regularMarketPrice, chartPreviousClose, volume
5. Calculate: change = price - prevClose, changePercent
6. Update stock in Supabase DB
7. Broadcast to Socket.io room "market-updates"
8. Frontend SocketContext receives 'stockPriceUpdate' event
9. Dashboard, Markets, StockDetails pages update in real-time
11 · Key Challenges & Solutions
Technical Problems Solved
⚡

Yahoo Finance Rate Limits

Problem: Concurrent requests for 50 stocks caused IP bans. Solution: Sequential processing with 1000ms delay between requests. Total cycle: ~50 seconds.

🤖

AI API Failures

Problem: Groq API socket hangups, timeouts. Solution: Dual-mode architecture — every AI endpoint has a local Node.js fallback with mock calculations.

🎯

Modal Positioning

Problem: OrderModal appeared at wrong position due to parent CSS transforms. Solution: React createPortal to render directly on document.body.

📐

Grid Card Stretching

Problem: Expanding one AI card stretched adjacent cards. Solution: Added items-start to CSS Grid container in AIInsights page.

🔄

SPA Page Refresh

Problem: Refreshing /login returned 404. Solution: Added vercel.json rewrites and _redirects for Netlify for proper SPA routing.

💰

Balance Sync

Problem: Cash balance not updating after trade. Solution: Frontend updateBalance() immediately updates local state, then refreshPortfolio() re-fetches from DB.
